DESCRIPCIÓN DE PROYECTO

1. Fase de pregunta

Este proyecto se implementó en una ferretería. El objetivo principal era promover las ventas a través de un programa de visitas a clientes. Visitar a los clientes en sus instalaciones permitiría detectar necesidades y finalmente obtener órdenes de compra.

2. Prepárese para la fase de exploración de datos

Primero, se necesitó implementar nuevos perfiles y folios en su ERP para dividir los documentos de ventas que son producto de este programa. Posteriormente se creó una aplicación para Android que comprueba si el vendedor realmente está visitando a los clientes. Lo anterior a través de su ubicación (ver imagen anexa).

3. Fase del proceso

Una vez que nos aseguramos de tener todos los datos que necesitamos para nuestro análisis y tenemos datos confiables y útiles, fue el momento de procesarlos. Primero, utilizamos Python y SQL para recuperar toda la información de ventas útil para el proyecto. En esta etapa filtramos documentos cancelados, consideramos impuestos y devoluciones, y corregimos errores de datos y tipos de datos.

4. Fase de análisis

Con la ayuda del director definimos cuántas visitas eran adecuadas para cada cliente, cuántas órdenes de compra deben obtener los proveedores y en qué plazo deben lograrlo.

5. Fase de compartir

Por último, los datos descritos y otros criterios se resumieron y representaron de forma amigable en un archivo Power Bi. En este informe podemos ver a través de una simple columna Verdadero/False, si el proveedor cumple con todas las metas establecidas para cada cliente. Esta información se visualiza en tiempo real. Puedes ver capturas de los programas generados en la sección de imagenes. Los datos mostrados fueron modificados por motivos de privacidad.

6. Fase de acción

La fase de acción fue realizada por el director, quien determinó reasignar los clientes a otros vendedores si no se alcanzan los objetivos. Arriba puede ver una página de este informe que ha sido modificada por motivos de privacidad.
