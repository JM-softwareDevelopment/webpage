DESCRIPCIÓN DE PROYECTO

1. Fase de pregunta

Este proyecto se implementó en una empresa de servicios de soporte. La empresa contaba con grandes problemas al no tener control de los servicios solicitados, tiempo de resolución y otra información importante.

2. Prepárese para la fase de exploración de datos

PPara resolver el problema, primero tuvimos que crear e implementar una página web de mesa de servicio. En esta etapa se definió el proceso a seguir y diseñamos la experiencia de usuario de la página web. Con la ayuda de JM se logró implementar el portal.

3. Fase del proceso

Se extrajeron los datos de la base de datos MySQL de la mesa de servicio. Se manipularon los datos para determinar si un ticket se retrasó considerando el Acuerdo de Nivel de Servicio (SLA) de la empresa.

4. Fase de análisis y compartir

Se creó un tablero operativo que muestra 3 puntos básicos: cuántas tareas están pendientes por realizar, quiénes son los responsables y qué tickets están fuera del SLA.

5. Fase de acción

Ahora se tienen datos suficientes para coordinar actividades y evitar un mal servicio, contratar gente cuando sea necesario, comprobar qué servicios son cruciales, formar a los técnicos en áreas relevantes, conocer mejor a los clientes, etc. Arriba puedes ver uno de los informes generados en este proyecto. Los datos fueron modificados por motivos de privacidad.
