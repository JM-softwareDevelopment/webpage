DESCRIPCIÓN DE PROYECTO

1. Fase de pregunta

Se detectó una necesidad del cliente de medir el comportamiento de sus cuentas por cobrar. El principal problema era identificar todos los ingresos que se recibirían en diferentes intervalos de tiempo. Con una mayor investigación detectamos que no contaban con métricas para evaluar el área de tesorería. Se desconocían temas importantes como los siguientes: cuánto cuesta invertir en otorgar créditos a clientes, cuánto tardan en pagar, cuántas veces en el año se han renovado las cuentas por cobrar y qué porcentaje de cuentas por cobrar se han recolectado. El objetivo de este proyecto era responder a todas estas preguntas con un informe sencillo.

2. Prepárese para la fase de exploración de datos

Para lograr el objetivo, se recolectó toda la información pertinente sobre los movimientos de las cuentas por cobrar en su sistema ERP. Para este proyecto se recolectó la siguiente información:

  • Importe total con y sin impuestos.
  • Período de crédito de los documentos de venta.
  • Fechas en que se produjeron las ventas y los pagos.
  • Clientes involucrados en cada movimiento.

El principal desafío en esta etapa es identificar la base de datos, tablas y campo que guardan la información que necesitamos para cumplir nuestros objetivos.

3. Fase de procesamiento

Usando SQL y Python se transformaron los datos para ordenarlos de una manera más significativa para nosotros. Las manipulaciones incluyeron:

  • Dinamización de datos.
  • Uniones.
  • Concatenación.
  • Agrupación de datos.
  • Restas, sumas y multiplicaciones.

Estas actividades tuvieron como finalidad obtener indicadores como:

  • Días de ventas pendientes.
  • Promedio de días en mora.
  • Índice de rotación de cuentas por cobrar.
  • Índice de efectividad de cobranza.

4. Fase de análisis

En este proyecto el cliente no requirió que JM se involucrará en el proceso de análisis de información. No obstante, se nos solicitó contribuir en la creación de un reporte para la representación de la información de forma amigable. Con ayuda de las herramientas de SQL, Python y Power Bi se creó el reporte que se visualiza en la sección de imágenes. Los datos mostrados fueron modificados por motivos de privacidad.
